# Ambient capture contract (1.1)

Normative behavior for project-scoped ambient recording. See also [`docs/plan/adoption-1.1.md`](plan/adoption-1.1.md) (A1).

## Decision order (`blackbox maybe-run`)

Evaluated in order; first match wins. **Passthrough never opens the store.**

| # | Condition | Action |
|---|---|---|
| 1 | `BLACKBOX_OFF` set | Passthrough bare command |
| 2 | `BLACKBOX_ACTIVE_RUN` set | Passthrough (nested under supervised run) |
| 3 | No enabled project config via shared discovery | Passthrough |
| 4 | Basename of argv[0] ∉ `[capture].wrap` | Passthrough |
| 5 | else | Record under discovered project store; set `BLACKBOX_ACTIVE_RUN` |

## Shell wrappers

Generated by `blackbox enable --install-shell`:

- Prefer `command blackbox maybe-run -- <name> …`
- If `blackbox` is missing from `PATH`, invoke bare `<name>` (never hard-fail the harness)
- Managed blocks use markers `# >>> blackbox >>>` … `# <<< blackbox <<<`
- Install is idempotent (single managed block)
- Uninstall removes only the managed block

## Escape hatches

```bash
export BLACKBOX_OFF=1          # this shell: no ambient capture
blackbox disable               # project: disable until re-enable
blackbox enable --uninstall-shell
```
